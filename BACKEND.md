## 4. Custom abstractions

### LocalizedTextField (`apps/core/fields.py`)
JSONB-backed Django field storing `{en: str, ru: str, ...}`. Default-factory returns `{"en": "", "ru": ""}`. Used on every localised content field per ADR-006.

### LocalizedSerializerField (`apps/core/serializers.py`)
DRF JSONField that ensures both `en` and `ru` keys are always present in API output, even when empty.

### RequestIDMiddleware (`apps/core/middleware.py`)
- Reads `X-Request-ID` from request or generates UUID4
- Attaches to `request.request_id` + contextvar
- Sets response header `X-Request-ID`
- Per KB-007, `get_request_id()` returns `""` (not `None`) outside request scope so log-record filter never raises

### CacheControlMiddleware (`apps/core/middleware.py`)
Per ADR-007 + KB-009: keys on `request.resolver_match.url_name` against `settings.CACHE_CONTROL_MAP` to apply `Cache-Control` + `Vary` headers. Single-source-of-truth caching policy.

### Custom DRF exception handler (`apps/core/exceptions.py`)
Emits the API_SPEC §5 envelope `{error: {code, message, details, request_id}}`. Per KB-010 uses `int(exc.wait or 0)` to handle `Throttled.wait == None` edge case + sets `Retry-After` header.

### HealthView (`apps/core/views.py`)
Throttle-exempt. DB ping via `connection.cursor().execute("SELECT 1")`. Sentry status check. Returns 503 when DB is down.

### Server-side PostHog mirror (`apps/core/posthog.py`)
Threading.Thread + queue.Queue background worker. Best-effort event capture — never raises, never blocks request thread. Activates only when `POSTHOG_SERVER_KEY` is set.

### Cloudflare CDN purge (`apps/core/cdn.py`)
`purge_charity(slug)` calls Cloudflare's purge_files API for the per-charity URLs. Gracefully no-ops when `CF_API_TOKEN` is missing. Wired via `Charity.post_save` signal in `apps/charities/signals.py` with 60s debounce to avoid hammering during bulk ETL.

## 5. ETL pipeline (per ADR-004)

**Full implementation**: `apps/ingestion/management/commands/ingest_propublica.py`

Flow per record:
1. `IngestionLog.objects.create(status=RUNNING)` opens audit row
2. `ThrottledHTTPClient.get()` — 5 req/sec self-imposed politeness
3. `_find_or_create()` — tiered dedup:
   - Hard match `(country, registration_id)` → reuse
   - Fuzzy `pg_trgm` ≥ 0.92 → auto-merge with audit log
   - 0.85–0.92 → flag for review (creates new but logs flag)
   - <0.85 → create new
4. `transaction.atomic` per record + `select_for_update(skip_locked=True)` on SourceMapping
5. `raw_data_hash` short-circuit (sha256 of canonicalised payload)
6. UPSERT Charity, Financial, SourceDocument; trigger updates `search_vector` + `name_trgm` automatically
7. Log `succeeded | partial | failed` to `IngestionLog`

**Stubs** (full implementation deferred — see §9):
- `ingest_every_org.py`
- `ingest_charitybase.py`

## 6. Caching strategy (per ADR-007)

| Layer | What | Where |
|---|---|---|
| Browser | `Cache-Control` headers per `url_name` | `CacheControlMiddleware` |
| Cloudflare CDN | Same `Cache-Control` honoured at edge; `s-maxage=3600` typical | DNS proxied at api.trustgive.org |
| Cachalot ORM cache | LocMem per Django worker; auto-invalidated on save; `CACHALOT_UNCACHABLE_TABLES` excludes ingestion + events | `settings/base.py` |

**Invalidation**: `Charity.post_save` signal calls `purge_charity(slug)` (debounced 60s). Catalog URLs rely on TTL only.

## 10. Railway deployment

`Dockerfile` is multi-stage (build deps separate from runtime). Final image runs:
```
gunicorn trustgive.wsgi:application --bind 0.0.0.0:$PORT --workers 2
```

Entrypoint script applies migrations + collects static before gunicorn starts.

`railway.json` (at project root, not inside `backend/`) tells Railway:
- Builder: `DOCKERFILE`, path `backend/Dockerfile`
- Healthcheck: `GET /api/health/` with 30s timeout
- Restart policy: `ON_FAILURE` with up to 5 retries

Postgres URL injected via Railway service reference: `DATABASE_URL = ${{Postgres.DATABASE_URL}}`.

## 11. Decisions diverged from spec/ADRs (with reason)

1. **`apps.i18n` AppConfig.label = "i18n_app"** — Django reserves `i18n` namespace; collision causes confusing INSTALLED_APPS resolution error. On-disk module path stays semantic. Documented in KB-BACKEND-TRUSTGIVE-006.

2. **`name_trgm` populated by SAME trigger as `search_vector`** — atomic with row writes; ADR-005 implied separate Python-side update. Single-trigger pattern (KB-BACKEND-TRUSTGIVE-005) avoids orphan-state bugs at ETL scale.

3. **Slug alias resolution returns 301 + JSON body `{redirect_to: ...}` + Location header** in viewset retrieve action — ADR-001 said "middleware-level". Doing it in the viewset lets the frontend optionally render a "this charity was renamed" interstitial while still satisfying HTTP semantics for crawlers.

4. **`raw_data_hash` is `BinaryField(max_length=32)`** — ADR-004 specified `bytea`. Same wire shape; Django's `BinaryField` is the conventional ORM mapping. No behavioural deviation.

5. **`CharityTrustBadge` and `SourceMapping` use `BigAutoField` PK** — both are M2M-through with no public-facing identity. Saves an index slot, faster joins, no API exposure.

6. **`get_request_id()` returns `""` (not `None`) outside request scope** — keeps the JSON log filter trivial and avoids `KeyError` when records are emitted from startup hooks or management commands.

7. **API_SPEC §10 reconciliation applied** in trigger SQL — reads JSONB `name->>'en'/'ru'` per ADR-006 storage. ADR-005 column-style reference superseded.

8. **`CACHALOT_UNCACHABLE_TABLES` uses Django db_table form** (`ingestion_ingestionlog`, NOT `ingestion.IngestionLog`) — KB-008. Wrong format silently no-ops.

9. **`CACHE_CONTROL_MAP` keyed on `url_name`** — KB-009. Every `path()` and `@action` has explicit `url_name=`.

10. **DRF custom exception handler uses `int(exc.wait or 0)`** — KB-010. Throttled.wait can be None on first throttle trip after cache reset.

## 12. Known limitations

- `ingest_propublica` field mappings for admin/fundraising expenses are best-effort (ProPublica's Form 990 schema varies by year). Phase 4.5 Performance Engineer + Testing Engineer will validate against real data and refine.
- ProPublica has no `updated_since` API parameter; nightly delta sync re-processes top-N records and relies on `raw_data_hash` short-circuit to elide identical re-imports.
- `CharitySlugAlias` 301 redirect happens at the API layer; for direct frontend navigation `https://trustgive.org/charities/{old-slug}/`, the Frontend Developer (Phase 4) will need to handle the 301 response or implement client-side redirect.
- No tests in this commit — Testing Engineer covers in Phase 4.5.
